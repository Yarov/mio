SYS://INFO

Mio is a persistent memory system for AI coding agents. It gives Claude Code the ability to remember decisions, discoveries, and context across sessions.

3 interfaces: MCP (15 tools), HTTP API (15 endpoints), TUI (mio tui).

FTS5 SearchFull-text search with temporal decay, importance boost, and frequency scoring.
Deduplication15-min window with normalized content hashing.
Topic UpsertsEvolving topics update in place via topic_key.
Relationssupersedes, relates_to, contradicts, builds_on, caused_by, resolved_by.
SessionsTrack work sessions with projects and summaries.
SyncChunk-based gzip export for multi-device sync.

SYS://ARCHITECTURE

How Mio's components connect. 4 interfaces feed into the central store, backed by SQLite with FTS5.

Interfaces
MCP Server (stdio)
15 tools. Claude Code communicates here. mem_save, mem_search, mem_context, mem_session_*, mem_relate...
HTTP REST API
15 endpoints. This dashboard + /search, /observations CRUD, /sessions, /stats, /export, /import.
Terminal UI (Bubbletea)
8 screens: dashboard, search, results, detail, timeline, sessions. Run mio tui.
CLI Commands
mio save/search/context/timeline/stats/export/import/sync. Direct terminal access.
STORE
sqlite + fts5
WAL mode
Temporal decay
Dedup hashing
Topic upserts
Auto-relations
6 tables + FTS
Busy timeout 5s
Data Layer
Observations
Core memories: title, content, type, project, importance, topic_key. FTS5 indexed.
Sessions
Work sessions with project, directory, start/end times, structured summaries.
Relations
6 types: supersedes, relates_to, contradicts, builds_on, caused_by, resolved_by.
Sync + Prompts
Chunk-based export with gzip. Prompt capture for replay. Manifest dedup.
SYS://SETUP

mio setup installs into Claude Code:

~/.claude/mcp/mio.jsonMCP server registration. Points to the mio binary.
~/.claude/settings.json15 tools added to permissions.allow + statusline config.
~/.claude/CLAUDE.mdMemory protocol injected. Proactive save, search, session rules.
~/.claude/skills/All SDD and coding skills copied here. Auto-detected by Claude.
~/.claude/statusline.shShows memory count and last search in status bar.
~/.mio/mio.dbSQLite database. Created on first run. All data lives here.

SDD://RULES
Scope AssessmentSmall (1-2 files): skip SDD. Medium (3-5): /sdd-ff. Large (6+): full pipeline.
Delegated ExecutionEach phase runs as a sub-agent with fresh context via the Agent tool. Orchestrator never codes.
Word Budgetsproposal: 400w, spec: 650w, design: 800w, tasks: 530w. Prevents context pollution.
Parallel Phasesspec + design run simultaneously after proposal. Both depend on proposal, not each other.
Recovery/sdd-continue detects state from Mio memory and resumes. Works across sessions.
Two-Step Retrievalmem_search returns previews only. Always call mem_get_observation(id) for full content.

ADMIN://ENDPOINTS

HTTP API reference — all endpoints available on this server.

GET /healthServer health check.
POST /observationsSave a new memory.
GET /observations/:idGet full observation.
PUT /observations/:idUpdate title/content.
DELETE /observations/:idSoft delete (?hard=true).
GET /search?q=...FTS5 search.
GET /contextRecent memories.
GET /timeline/:idTimeline around obs.
GET /relations/:idRelated observations.
POST /sessionsStart new session.
PUT /sessions/:id/endEnd session.
GET /sessionsList sessions.
GET /statsMemory metrics.
GET /exportExport all as JSON.
POST /importImport from JSON.
GET /skillsList installed skills.
GET /skills/:nameRead skill content.
PUT /skills/:nameUpdate skill content.
GET /agentsList all agents and status.
POST /admin/setup?agent=...Setup Mio for agent.
POST /admin/uninstall?agent=...Remove Mio from agent.
ADMIN://CLI
mio setup [agent]Configure Mio for an agent (default: claude-code). --all for all detected.
mio setup --listShow all supported agents and their status.
mio uninstall [agent]Remove Mio from an agent. --all for all. --purge to delete data.
mio serve [port]Start this HTTP dashboard + API server.
mio tuiLaunch interactive terminal UI.
mio mcpStart MCP stdio server (used by Claude Code).
mio syncExport/import chunks for multi-device sync.
